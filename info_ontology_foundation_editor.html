This editor is in an experimental state

When started and defining a new ontology, the required directory is generated and a basic ontology file is loaded. The elements defined in this skeleton ontology are used later in the program set. These fixed parts are currently not protected.



Tree structure

Branches can be related via inter or intra.

Inter defines the two branches to be connected over interfaces not transferring tokens but information, while intra imposes intrafaces transferring tokens.



Section structure

nodes: (currently in the initial file)

root > node > constant > lumped
root > node > dynamic > lumped
root > node > dynamic > distributed
root > node > event > lumped
root > node > event > distributed

tokens :

for each token transport a mechanism and a nature must be defined
